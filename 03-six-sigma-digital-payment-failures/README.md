# Six Sigma Digital Payment Failure Project

## Project Overview
This project applies the Six Sigma DMAIC method to analyse and reduce digital payment failures in a simulated banking app environment.

The project focuses on identifying the main causes of failed digital transactions, prioritising improvement opportunities using Pareto analysis, identifying root causes through 5 Whys analysis and proposing practical control measures.

## Business Problem
The banking app was experiencing failed digital transactions, which could affect customer trust, user experience, complaint levels and transaction success rates.

## Project Goal
Reduce digital payment failures by 40% within 6 months by identifying and addressing the main root causes of failed transactions.

## My Role
Business Analyst / Process Improvement Analyst

## Method Used
Six Sigma DMAIC:
- Define
- Measure
- Analyse
- Improve
- Control

## Tools Used
- Excel
- PivotTables
- Pareto analysis
- 5 Whys root cause analysis
- KPI monitoring plan
- Process improvement planning

## Dataset Used
The project used a mock digital payment transaction dataset containing 1,000 transactions with fields such as Transaction ID, Timestamp, Amount, Payment Method, Bank Name, Status and Failure Reason.

## Key Findings
The analysis found 222 failed transactions, giving a failure rate of 22.2%. The top three failure reasons were User Cancelled, Insufficient Funds and Bank Server Timeout, accounting for approximately 55.8% of total failures.

## Recommendations
- Add real-time progress indicators and confirmation screens
- Add pre-payment balance validation
- Introduce automatic retry logic for timeout errors
- Monitor failure rate, timeout errors and customer feedback after implementation
