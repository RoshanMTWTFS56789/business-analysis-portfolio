# Retail Order Fulfilment Improvement Project

## Project Overview
This project is an end-to-end Business Analysis and analytics case study for a simulated omnichannel retailer experiencing fulfilment, stock-out and OTIF performance issues.

The project demonstrates how Business Analysis can connect business problem definition, stakeholder needs, process improvement, KPI design, Excel analysis, Power BI dashboarding, Agile delivery and AI-assisted decision support.

## Business Problem
The retailer had reasonably strong demand forecasting, but fulfilment performance remained inconsistent. The analysis found that the main issue was not forecasting accuracy alone, but supplier under-delivery, stock-out risk, weak replenishment controls and limited exception visibility.

## Key Insight
Forecast accuracy alone does not guarantee fulfilment success. The bigger improvement opportunity was in supplier reliability, replenishment control and exception-based management.

## Recommendations
- Introduce rule-based replenishment controls
- Monitor supplier reliability
- Use exception-based management
- Apply AI as decision support, not full automation
- Build Power BI dashboards for executive and operational visibility
